## 팀 미팅 및 작업

### 1. HTTPS 설정 완료 및 개발 환경 동기화

오늘 회의에서는 민재 님이 프론트엔드와 백엔드의 HTTPS 설정을 완료했다는 중요한 업데이트를 공유했습니다. 이제 모든 팀원이 로컬 개발 환경에서도 실제 배포 환경과 유사하게 HTTPS 기반으로 테스트하고 개발할 수 있게 되었습니다. (Chocolatey 설치 및 인증키 발급 필요) 보안된 환경에서 개발을 진행할 수 있게 되어 안정성이 높아졌습니다.

### 2. 문서 및 기능 개발 현황 공유

각 팀원의 문서 및 기능 개발 현황을 공유했습니다. 관주 님은 REST API 문서 작성을 완료했고, 저는 어제 맡았던 **사용자 플로우, ERD(논리/개념), WebSocket API 명세서 작성을 완료**했음을 보고했습니다. 동준 님은 실시간 수어 퀴즈 UI를 완료했고, 승현 님은 FastAPI와 프론트엔드 연결 작업을 진행 중입니다. 주요 기능 개발과 문서화가 병행되며 체계적으로 진행되고 있음을 확인했습니다.

### 3. 실시간 퀴즈 프론트엔드 개발 분담

실시간 퀴즈 UI와 HTTPS 설정이 완료됨에 따라, 본격적인 프론트엔드 연동 작업을 시작하기로 했습니다. 논의 결과, **관주 님은 게임 시작 전 로직(방 생성, 입장, 대기 등)을, 저는 게임 시작 후 로직(퀴즈 진행, 점수 표시, 결과 등)**의 프론트엔드 개발을 맡기로 결정했습니다. 백엔드 로직을 직접 구현했던 만큼, 해당 로직을 프론트엔드 화면에 연결하는 작업을 직접 맡게 되어 효율적인 개발이 기대됩니다.

## 개인 작업

### 1. 주요 설계 문서 작성 완료

* **진행 내용**: 어제 액션 아이템이었던 **사용자 플로우, 개념/논리 ERD, WebSocket API 명세서 작성을 모두 완료**했습니다. 백엔드 퀴즈 로직 구현 내용을 바탕으로 최신 상태를 반영하여 문서들을 업데이트했습니다. 이를 통해 팀원들이 현재 시스템 구조와 데이터 흐름, API 사용법을 명확하게 이해하고 개발을 진행하는 데 도움이 될 것으로 기대합니다.

### 2. 실시간 수어 퀴즈 프론트엔드 개발 착수 (게임 시작 후 로직)

* **진행 내용**: 오늘 회의에서 결정된 대로, **실시간 수어 퀴즈의 프론트엔드 개발**에 착수했습니다. 제가 담당할 부분은 게임이 시작된 이후의 화면과 로직으로, WebSocket을 통해 백엔드로부터 받는 게임 상태(문제, 도전자, 점수, 타이머 등)를 화면에 표시하고, 사용자 인터랙션(정답 제출 등)을 처리하는 부분을 구현하기 시작했습니다.

## 느낀점

어제 맡았던 문서 작업들을 오늘 모두 마무리하고 나니 마음이 한결 가볍습니다. 특히 제가 직접 구현한 백엔드 로직을 바탕으로 문서를 최신화하는 과정에서 시스템 전체 구조를 다시 한번 정리할 수 있었습니다. 문서화가 단순히 기록을 남기는 것을 넘어, 구현 내용을 명확히 하고 잠재적인 개선점을 발견하는 데도 도움이 된다는 것을 느꼈습니다.

이제 백엔드 개발을 넘어 프론트엔드 개발까지 담당하게 되니, 풀스택 개발자로서의 역량을 키울 좋은 기회라는 생각이 듭니다. 백엔드에서 설계한 데이터 흐름이 프론트엔드 화면에서 어떻게 시각적으로 구현되고 사용자와 상호작용하는지를 직접 경험하면서 더 깊이 있는 이해를 할 수 있을 것 같습니다. HTTPS 설정까지 완료되었으니, 이제 본격적으로 실제 게임 화면을 만들어나갈 생각에 기대가 됩니다.
